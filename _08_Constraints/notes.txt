Constraints
    SQL constraints are used to spwcify rules for data in data
    1. NOT NULL  - columns can't have null value (e.g. col1 int NOT NULL)
    2. UNIQUE - all values in column are different  (e.g. col1 int UNIQUE)
    3. PRIMARY KEY - A Primary Key is a column (or combination of columns) that uniquely identifies each row in a table.
        3.1 Single Column Primary Key :
                CREATE TABLE students (
                    id INT PRIMARY KEY,
                    name VARCHAR(50),
                    city VARCHAR(50)
                );
        3.2 Composite Primary Key (Multiple Columns) :
                CREATE TABLE temp (
                    id INT,
                    name VARCHAR(50),
                    city VARCHAR(20),
                    PRIMARY KEY (id, name)  // It means same id with same name not allowed
                );
    4. FOREIGN KEY - Prevent actions that would destroy link between tables,
        CREATE TABLE temp(
            cust_id int,
            FOREIGN KEY (cust_id) reference customer(id)
        )
        here, cust_id = new table id and id = column of existing table
    5. DEFAULT - Sets the default value of a column
        salary INT DEFAULT 250000
    6. CHECK - It can limit the values allowed in a column
        e.g.    CREATE TABLE city (
                    id INT PRIMARY KEY,
                    city VARCHAR(50),
                    age INT,
                    CONSTRAINTS age_check CHECK(age>=18 AND city= "Delhi)
                );
                        OR
                CREATE TABLE city (
                    age INT CHECK (age>=18);
                )
